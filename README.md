> **项目描述：** 基于 UE5 开发的具有完整战斗循环的射击原型，实现了高自动化的 AI 战斗逻辑与细腻的角色动画表现。

* **🏃 深度集成的动画系统 (Animation System)**

    * **状态机架构 (State Machine)：** 建立复杂的动画状态机，精准处理 `移动`、`瞄准`、`死亡` 等状态的逻辑切换与动画融合。

* **🎯 精准的射线探测射击判定 (Line Trace Interaction)**

    * **全沉浸式反馈：** 集成 **Niagara 粒子系统**（枪口火光、烟雾）与声音系统，构建从视觉到听觉的打击感闭环。
#### 2️⃣ [坦克大战| Tank Battle (C++ / UE5)](https://github.com/18109508757junjun-dot/UE5C-BattleBlaster)
> **项目描述：** 这是一个侧重于 C++ 类层级设计模拟战斗项目。通过抽象基类实现了玩家控制坦克与 AI 自动化防御塔的逻辑复用。

* **🏗️ 充分利用C++继承特性，合理的游戏架构 (Polymorphic Architecture)**
    * **设计思路：** 抽象出 `ABasePawn` 作为核心基类，统一封装了炮塔旋转逻辑、弹药生成位点以及 `Fire`（开火）的虚函数接口。
    * **实际应用：** 派生出 `ATank`（处理玩家输入与位移）与 `ATower`（处理 AI 自动追踪），展示了对 **OOP（面向对象）** 中“高内聚、低耦合”原则的深度实践。

* **📐 物理插值与手感调优 (Physics Smoothing & Math)**
    * **技术实现：** 针对炮塔转向，弃用硬连线的角度设置，改用 `FMath::RInterpTo` 算法结合 `WorldDeltaSeconds` 进行动态插值。
    * **解决痛点：** 有效解决了在不同帧率下旋转速度不一致的问题，实现了丝滑且具有物理质量感的准星追踪效果。

* **⏱️ 基于定时器的射击与 AI 管理 (Timer-based Logic)**
    * **性能优化：** 弃用高开销的 `Tick` 轮询，利用 `FTimerHandle`（定时器句柄）精确管理 AI 的射击频率和玩家检测距离。
    * **逻辑闭环：** 实现了“范围检测 -> 锁定目标 -> 计时器触发开火”的完整 AI 逻辑链条，大幅降低了 CPU 在大规模单位场景下的计算负载。

* **💣 物理弹药系统与伤害处理 (Projectile & Damage)**
    * **物理模拟：** 独立开发 `AProjectile` 类，利用 `ProjectileMovementComponent` 实现具有真实弹道的物理模拟。
    * **碰撞反馈：** 处理 `OnComponentHit` 事件，调用 `UGameplayStatics::ApplyRadialDamage` 实现范围伤害，并集成爆炸粒子与相机震动反馈。

#### 3️⃣ [组件化地牢交互系统 | Dungeon Escape (C++ / UE5)](https://github.com/18109508757junjun-dot/UE5C-dugeonescape-)
> **项目描述：** 一套基于“触发-响应”机制的模块化解谜框架。通过 C++ 自定义组件实现了逻辑的高度解耦，支持在编辑器中快速构建复杂的机关联动。

* **🧩 模块化组件设计 (Component-Based Architecture)**
    * **设计思路：** 开发通用的 `UMover`（移动组件）与 `UTriggerComponent`（触发组件），实现“功能与物体分离”。
    * **技术优势：** 任何 Actor（如大门、升降梯、暗门）挂载 `UMover` 即可获得平滑位移能力，无需针对每个物件重复编写逻辑，极大提升了资产复用率。

* **🔗 触发器组件动态关联 (Dynamic Component Discovery)**
    * **技术实现：** 利用 `FindComponentByClass` 动态获取关联组件，实现触发器与受控物体间的弱耦合通信。
    * **解耦表现：** 触发组件仅负责发送逻辑信号（开启/关闭），无需关注受控物体的具体类型，支持关卡设计师在编辑器内自由组合机关逻辑。

* **🏃 平滑运动控制 (Interp Movement Control)**
    * **核心算法：** 运用 `VInterpConstantTo` 结合 `DeltaSeconds` 实现帧率无关（Frame-rate Independent）的匀速运动。
    * **逻辑细节：** 精确控制物体的起始坐标与目标偏移，支持多状态平滑过渡，确保在不同性能硬件下视觉表现的一致性与丝滑感。

* **🔍 物理探测与交互判定 (Physics Tracing)**
    * **技术方案：** 封装基于 `SphereTrace` 或 `BoxOverlap` 的实时重叠检测系统，支持多层级属性过滤（Actor Tags）。
    * **交互逻辑：** 实现了精准的物体拾取判定与机关激活检测，为解谜玩法（如：特定钥匙球激活对应基座）提供了稳定的底层支撑。

#### 5️⃣ [障碍挑战 Assault 系统 | Moving Obstacles (C++ / UE5)](https://github.com/18109508757junjun-dot/UE5C-ObstacleAssault)
> **项目描述：** 侧重于游戏物体的程序化运动控制与向量数学应用，通过纯 C++ 算法确保复杂物理机关在不同硬件环境下的运动同步。

* **🔄 程序化运动算法 (Procedural Movement Control)**
    * **技术实现：** 编写通用的障碍物组件逻辑，通过 C++ 实现 Actor 的**往复直线运动、环形轨道移动**以及**复合自转逻辑**。

    * **运动曲线：** 利用三角函数（Sin/Cos）构建平滑的往复运动曲线，有效消除了机械运动在转向点的视觉生硬感。

    * **物理交互：** 解决了移动平台与玩家角色（Pawn）之间的摩擦力补偿与位置同步问题，确保了高频移动下的碰撞稳定性。
